# 平衡车 PID 调参指南

---

## 当前状态

所有硬件驱动正常，MPU6050 轴已校准，机械零点 ≈ 0°。小车能响应倾倒，但回正过于剧烈。

**当前参数：** [control.c](Template/HARDWARE/CONTROL/control.c)

```c
#define MECHANICAL_ZERO  2.3f   /* 机械零点 — 如果直立时 pitch!=0，修改此值 */
float Balance_Kp = 200.0f;      /* 直立环比例 ← 当前过大 */
float Balance_Kd = 0.5f;        /* 直立环微分 */
int   Velocity_Kp = 50;         /* 速度环比例 */
float Velocity_Ki = 0.0f;       /* 速度环积分 — 先保持 0 */
```

改完参数后 **F7 编译 → F8 烧录**，无需改动任何其他文件。

---

## 调参总原则

```
先内环（直立 PD），后外环（速度 PI）
每次只改一个参数，观察效果后再改下一个
```

---

## 第一步：直立环 PD（核心）

### 1.1 先降 Balance_Kp

当前 200 太大导致剧烈抖动。**每次减半**直到出现低频无力：

| 步骤 | Balance_Kp | 预期现象 |
|------|-----------|---------|
| 1 | **100** | 抖动减轻，可能仍偏强 |
| 2 | **60** | 较柔和，可能有小幅度来回震荡 |
| 3 | **40** | 感觉偏软，松手站 1-2 秒就倒 |

找到"站不住"和"抖动"之间的值，取中间值。

### 1.2 再调 Balance_Kd

Kd 抑制震荡，让车变"硬"。在 Kp 定好后逐步增大：

| Balance_Kd | 预期现象 |
|-----------|---------|
| 0.3 | 车软绵绵，来回晃 |
| 0.5 | 略硬，有抑制效果 |
| 0.8 | 可能高频抖动（电机滋滋响） |
| 1.0+ | 反应迟钝，推不动 |

找到刚好不抖的值，再乘以 0.7 作为最终值。

### 1.3 预期效果

调好直立环后，小车应该能**松手站立 3-5 秒**不倒，轻推能回正。

---

## 第二步：速度环 PI

直立环稳定后再加。速度环让小车"保持在原地"。

### 2.1 Velocity_Kp

| Velocity_Kp | 预期现象 |
|------------|---------|
| 0 | 车能站，但慢慢漂移 |
| 20 | 有拉回趋势 |
| 50 | 可能推一下能回位 |
| 80+ | 来回拉锯，前后晃动 |

找到推车能缓慢回位、不自激震荡的值。

### 2.2 Velocity_Ki（最后调）

消除静差，让车真正停住。初始值建议：

```
Velocity_Ki = Velocity_Kp / 200.0f
```

例如 Kp=60 → Ki=0.3。Ki 太大会积分饱和导致翻车，**宁小勿大**。

---

## 典型参数参考

| 参数 | 温柔 | 适中 | 激进 |
|------|------|------|------|
| Balance_Kp | 40~60 | 80~150 | 200+ |
| Balance_Kd | 0.3~0.5 | 0.5~0.8 | 1.0+ |
| Velocity_Kp | 20~40 | 40~80 | 100+ |
| Velocity_Ki | Kp/200 | Kp/200 | Kp/100 |

## 现象诊断速查

| 现象 | 原因 | 调整 |
|------|------|------|
| 剧烈高频抖动，嗡嗡响 | Kp 或 Kd 过大 | 先减 Kp，不减再减 Kd |
| 低频来回荡，像不倒翁 | Kd 太小 | 增大 Kd |
| 站不住，一碰就倒 | Kp 太小 | 增大 Kp |
| 站得住但往一个方向跑 | 机械零点不准 | 修正 MECHANICAL_ZERO |
| 站得住但越跑越快 | 缺速度环 | 增大 Velocity_Kp |
| 推车不回来 | Velocity_Kp 太小 | 增大 Velocity_Kp |
| 前后拉锯震荡 | Velocity_Kp 太大 | 减小 Velocity_Kp |
| 车不动/没反应 | 按 KEY2(PA5) 开启电机 | Motor 显示 ON |
| 朝一个方向猛冲 | 陀螺轴配错 | 检查 exti.c 中 ax/ay/gx/gy 组合 |

---

## 每次测试流程

```
1. 修改 control.c 中的参数
2. F7 编译 → F8 烧录
3. 小车放地上扶正
4. 按 KEY2 开启电机（OLED 显示 Motor: ON）
5. 松手观察
6. 出现异常立即扶住
```

## ADC 电压校准（待做）

当前电池电压显示不准，低压保护已暂时绕过。校准步骤：

1. 用万用表测量电池实际电压 X.XX V
2. 串口或 OLED 读取当前显示的 battery_voltage = Y.YY V
3. 在 [adc.c](Template/Drivers/BSP/ADC/adc.c) 中修改分压比：

```c
float volt = (float)adc_val * 3.3f * 43.0f / 4096.0f / 10.0f;
//                                  ^^^              ^^^
//                            R1+R2 总电阻       R2 下电阻
```

4. 实际分压比 = X.XX / Y.YY * (43.0/10.0)
5. 修改公式后恢复低压保护：在 [exti.c](Template/Drivers/BSP/EXTI/exti.c) 中取消注释：
   ```c
   balance_enable = (battery_voltage >= 10.0f) ? 1 : 0;
   ```
