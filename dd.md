# STM32 平衡小车烧录与调参指南

---

## 一、编译产物烧录

### 方法一：Keil MDK 内置下载器（推荐）

1. **硬件连接**
   - 将 ST-Link 下载器连接到电脑 USB
   - ST-Link 引脚 ↔ 平衡小车：
     ```
     ST-Link SWDIO → 小车 SWDIO
     ST-Link SWCLK → 小车 SWCLK
     ST-Link GND   → 小车 GND
     ST-Link 3.3V  → 小车 3.3V（或电池供电）
     ```

2. **配置 Keil 下载器**
   - 点击菜单 `Project` → `Options for Target`（或按 Alt+F7）
   - 切换到 `Debug` 选项卡
   - 右侧下拉选择 `ST-Link Debugger`
   - 点击 `Settings`
     - `Port` 选择 `SW`（串行线调试）
     - `Max Clock` 设为 4MHz
     - 确认 `SW Device` 中识别到 `STM32F103C8`

3. **一键下载**
   - 点击 Keil 工具栏 `Load` 按钮（或按 F8）
   - 底部输出窗口显示：
     ```
     Load "final.axf" 
     Erase Done.
     Programming Done.
     Verify Done.
     ```
   - ✅ 烧录完成

### 方法二：STM32 CubeProgrammer

1. 安装 STM32 CubeProgrammer
2. 连接 ST-Link，选择 `USB` 模式，点击 `Connect`
3. 点击 `+` 选择 `Template/final.hex`
4. 点击 `Download`

### 方法三：串口 ISP 下载

- 使用 CH340 USB-TTL 模块，BOOT0 接高电平，按复位键进入 ISP 模式
- 使用 `FlyMcu.exe` 或 `Flash Loader Demonstrator` 下载 hex 文件

## 二、串口调试输出

### 硬件连接
```
USB-TTL TX → 小车 PA10 (USART1 RX)
USB-TTL RX → 小车 PA9  (USART1 TX)
USB-TTL GND → 小车 GND
```

### 串口助手设置
| 参数 | 值 |
|-----|-----|
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 换行符 | LF 或 CRLF |

### 输出格式
```
P=12.3 G=-0.5 V=7.4 S=0 E=1
│     │       │       │    └─ 平衡使能 (1=开启)
│     │       │       └─────── 左右轮编码器速度和
│     │       └─────────────── 电池电压 (V)
│     └─────────────────────── 陀螺仪角速度 (°/s)
└───────────────────────────── 车身倾角 (°)
```

## 三、PID 参数调参指南

### 📍 参数文件位置
`Template/HARDWARE/CONTROL/control.c`

```c
/* 直立环 PD 参数 */
float Balance_Kp = 200.0f;   // 直立环比例系数
float Balance_Kd = 0.5f;     // 直立环微分系数

/* 速度环 PI 参数 */
int Velocity_Kp = 50;        // 速度环比例系数
int Velocity_Ki = 0;         // 速度环积分系数
```

---

### 🎯 第一步：直立环 PD 参数（核心！）

**机械中立点校准**
1. 将小车放在水平桌面上，手扶正使其直立
2. 观察串口输出 `P=X.X`，记录静止时的倾角值（例如 0.2°）
3. 这就是你的机械零点，后续调参以此为基准

**Balance_Kp 调整步骤：**

| Kp 值 | 现象 | 调整方向 |
|-------|------|---------|
| 太小 | 小车无力，轻轻一碰就倒，震荡缓慢 | ⬆ 增大 |
| 太大 | 高频抖动，车身剧烈振动，嗡嗡作响 | ⬇ 减小 |
| 合适 | 松手后能站立 2-3 秒，摇晃频率适中 | ✓ 保留 |

**Balance_Kd 调整步骤：**
> Kd 相当于"阻尼"，抑制 Kp 带来的震荡

| Kd 值 | 现象 | 调整方向 |
|-------|------|---------|
| 太小 | 来回震荡，像不倒翁一样晃来晃去 | ⬆ 增大 |
| 太大 | 反应迟钝，一碰就倒，电机滋滋响 | ⬇ 减小 |
| 合适 | 轻轻推一下，能迅速回正，无余振 | ✓ 保留 |

**✅ 直立环调参口诀：**
> 先调 Kp 到低频震荡 → 再调 Kd 消震荡 → 循环往复直到直立

---

### 🎯 第二步：速度环 PI 参数

> 直立环调好后再加速度环！否则车会乱跑

**Velocity_Kp 调整步骤：**
> 速度环让小车"保持在原地"

| Kp 值 | 现象 | 调整方向 |
|-------|------|---------|
| 太小 | 推车不回位，越跑越远 | ⬆ 增大 |
| 太大 | 前后剧烈震荡，像拉锯 | ⬇ 减小 |
| 合适 | 推车后缓慢回原点，无震荡 | ✓ 保留 |

**Velocity_Ki 调整步骤：**
> Ki 消除静差，让车真正停住

- 初始值设为 `Velocity_Kp / 200`，例如 Kp=50 → Ki=0.25
- Ki 不要太大，否则会积分饱和导致翻车

---

### 🎯 第三步：联调优化

两个环会相互影响，需要反复微调：

1. **现象：小车静止时直立很好，一走就倒**
   - → 减小 Velocity_Kp，速度环不要太强
   
2. **现象：小车直立还行，但慢慢往一个方向跑**
   - → 检查机械零点是否偏移
   - → 适当增大 Velocity_Ki

3. **现象：开机抖动几下然后倒下**
   - → Balance_Kd 太大了
   - → 或电池电压不足

## 五、常见问题

### ❌ 烧录失败
- 检查 ST-Link 接线是否牢固
- 确认供电充足（电池或 ST-Link 3.3V）
- Keil Debug 设置中确认识别到芯片

### ❌ 电机不转
- 检查 KEY2 是否已按下（PA5 按键切换电机使能）
- 检查电池电压（低于 6.5V 欠压保护）
- 串口看 `Motor: ON` 还是 `OFF`

### ❌ 车一开机就往一个方向猛冲
- 机械零点偏差太大
- MPU6050 安装方向是否正确（Z 轴向上）
- Balance_Kp 极性反了（加个负号试试）

### ❌ 烧录后没反应
- 检查 BOOT0 引脚是否接回 GND
- 按一下复位键

2. **低通滤波系数**（减小噪声影响）
3. **死区补偿**（消除电机 PWM 死区）
